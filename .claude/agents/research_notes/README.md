# Research Notes

These files are supporting notes for implementation tasks.

They are not the source of truth.

Use this precedence order:

1. `PROMPT.md`
2. assigned task file
3. `AGENTS.md`
4. `CLAUDE.md`
5. research notes in this folder

Every research note should make these things explicit:

- what is verified from primary or official sources
- what is only a benchmark hypothesis
- what still needs project-specific testing
- which implementation decision it affects

If a research note conflicts with `PROMPT.md`, follow `PROMPT.md` and update the note in a later task.
